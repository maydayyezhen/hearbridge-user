## 第二步：统一路由真源

### 已完成改动

调整：

- `router/RoutePatternRegistry.ets`
- `router/RouteMatcher.ets`
- `router/RouteRenderer.ets`

### 实际整改内容

1. `RoutePatternRegistry.ets` 不再手写第二份路由表，改成从 `RouteRegistry` 派生：
   - `ROUTE_PATTERNS = ROUTES.map(...)`
   - `ROOT_TAB_PATHS = ROOT_TABS.map(...)`

2. `RouteMatcher.ets` 不再依赖手写 patterns 文件作为唯一来源，改为直接从 `ROUTES` 派生匹配目标。

3. `RouteRenderer.ets` 删除自己那套重复匹配思路，改为：
   - 先 `matchRoute(path)`
   - 再在 `ROUTES` 中找到命中项并执行 `render()`

### 自检结果

- 路由模式已改成**派生数据**，不再是第二份手写真源。
- `RouteRenderer.ets` 已复用 `matchRoute(path)`。
- `RouteMatcher.ets` 已直接依赖 `RouteRegistry`。

自检证据：

- `router/RoutePatternRegistry.ets` 中存在 `ROUTES.map((item) => item.path)`
- `router/RouteMatcher.ets` 中存在 `import { ROOT_TABS, ROUTES } from './RouteRegistry'`
- `router/RouteRenderer.ets` 中存在 `const matchedPath = matchRoute(path)`
