# utils 目录说明

本目录存放跨模块通用工具函数。

## 职责
- 提供不适合归属到单一业务模块的纯工具函数。
- 存放格式化、映射、兜底、纯计算类能力。

## 约定
- `utils` 不负责编排业务流程。
- 如果某个工具明显只服务单一模块，应优先回收到对应模块内。
- 与历史兼容相关的纯规范化函数，可以优先收口在这里或对应 service 中，而不是留在 store。


## 当前收口说明
- 仅服务记录页的映射能力已回收到 `services/record/mapper`，避免继续以通用 `PageMappers` 的名义挂在 `utils` 下制造误导。

- `TrainingTypeUtil.ets`：统一处理训练类型、练习模式、训练项状态这类历史兼容与归一化值。
